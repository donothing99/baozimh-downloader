# baozimh-downloader

A universal downloader for BaoZi Manga (baozimh.org). Download any manga from the site with all chapters preserved in order.

[中文文档](#中文文档)

## Features

- **Universal**: Works with any manga on baozimh.org, just provide the slug
- **Page-level resume**: Already downloaded pages are skipped; re-run to continue from where it stopped
- **Multi-CDN fallback**: Tries multiple CDN nodes for higher success rate
- **Chapter range**: Download a specific range of chapters
- **Auto-decrypt**: Executes the site's original decoder via Node.js, adapts automatically when the site updates
- **Decrypt retry**: Retries decryption on failure (decoder is fetched from network each time)
- **Chapter listing**: List chapters without downloading
- **Log to file**: Optional `--log` to output logs to a file
- **Clean exit**: Ctrl+C is handled gracefully without stack traces

## Requirements

- Python 3.7+
- Node.js 18+ (built-in fetch API required)
- requests library

## Setup

```bash
pip install -r requirements.txt
```

## Usage

### Basic

```bash
# Download a manga (saves to ./<manga_title>/)
python download.py fannaoxiyouji

# Download to a specific directory
python download.py fannaoxiyouji -o /path/to/output

# Download only chapters 5-10
python download.py fannaoxiyouji --start 5 --end 10

# List chapters only
python download.py fannaoxiyouji --list

# Force re-download (don't skip existing pages)
python download.py fannaoxiyouji --no-skip

# Save logs to a file (still outputs to console)
python download.py fannaoxiyouji --log download.log
```

### How to get the manga slug

The slug is the identifier in the manga's URL. Open the chapter list page:

```
https://m.baozimh.org/chapterlist/fannaoxiyouji
                                  ^^^^^^^^^^^^^^
                                  this is the slug
```

### Full arguments

```
positional arguments:
  slug                  manga slug (e.g. fannaoxiyouji)

optional arguments:
  -o, --output          output directory (default: ./<manga_title>)
  --start N             start chapter number (1-based)
  --end N               end chapter number (inclusive)
  --list                list chapters only, don't download
  --no-skip             don't skip already downloaded pages
  --log FILE            output logs to FILE (still prints to console)
```

### Output structure

```
<manga_title>/
  001_Chapter 1/
    001.webp
    002.webp
    ...
  002_Chapter 2/
    001.webp
    002.webp
    ...
```

## How it works

1. Extracts the manga ID (mid) from the chapter list page
2. Calls the API to get the full chapter list and encrypted image data for each chapter
3. Decrypts image URLs by executing the site's original `chapter-decoder.js` via Node.js (with retry)
4. Downloads images from CDN, saved in chapter and page order
5. Page-level resume: partially downloaded chapters continue from the first missing page

## Files

```
download.py          Main script (Python)
decode_helper.js     Image URL decryption helper (Node.js)
requirements.txt     Python dependencies
.gitignore           Git ignore rules
README.md            This file
```

`decode_helper.js` must be in the same directory as `download.py`.

## Notes

- The decoder script is fetched from the site (~21KB) each time to stay up-to-date; decryption failures are retried 3 times
- Image CDN has no referer check, images can be downloaded directly
- 1-second delay between chapters to avoid rate limiting
- If interrupted (Ctrl+C), re-run the script; already downloaded pages will be skipped automatically
- Partially downloaded chapters are resumed from the first missing page (not skipped entirely)

# 中文文档

包子漫画 (baozimh.org) 通用下载器，支持下载站点上任意漫画的全部章节。

## 特性

- **通用**: 支持包子漫画上任意漫画，只需提供slug
- **页级断点续传**: 已下载的页面自动跳过，中断后从缺失页继续（非整章跳过）
- **多CDN回退**: 自动尝试多个CDN节点，提高下载成功率
- **选话下载**: 支持指定章节范围
- **自动解密**: 通过Node.js执行原始解密器，网站更新解密逻辑后自动适配
- **解密重试**: 解密失败自动重试（解密器每次从网络加载，可能因网络波动失败）
- **章节列表**: 支持只查看章节列表不下载
- **日志文件**: 支持 `--log` 参数输出到文件
- **优雅退出**: Ctrl+C 干净退出，不打印堆栈信息

## 工作原理

1. 从章节列表页提取漫画ID (mid)
2. 调用API获取全部章节列表和每话的加密图片数据
3. 通过 Node.js 执行站点原始的 `chapter-decoder.js` 解密图片URL（带重试）
4. 从CDN下载图片，按章节和页码顺序保存
5. 页级续传：部分下载的章节从缺失的第一页继续，而非跳过整章

## 注意事项

- 解密时每次从站点下载 `chapter-decoder.js`（约21KB），可能因网络波动失败，已内置3次重试
- 图片CDN无防盗链，可直接下载
- 章节间有1秒延迟，避免请求过快
- Ctrl+C 中断后重新运行，已下载的页面会自动跳过
- 某章只下了一半不会整章跳过，会从缺失页继续下载
